Preparing the Device

Before doing anything, Install all the pre-requisites properly

Please follow the steps, in proper order, for successful installation

  1. Open SQL Server Management Studio 2012(may be named as SQL Management Studio)

    • If it didn’t open or, isn’t present, your installation of SQL Server 2012 Enterprise is improper.
      • Re-install SQL Server 2012 Enterprise and select these relevant options
        Click here to view Step-by-step walkthrough

        This walkthrough will guide you through the proper installation steps

        1. On the pre-requisites page, you will find link to download SQL Server 2012 Enterprise. Please secure it.
        2. Extract the ZIP file and start installing.
        3. You will reach this page first.
          • Select , the first option, New SQL Server stand-alone installation or add features to an existing installation
        4. On the next screen, the setup will scan fore relevant updates. This is optional. If your target setup device isn’t connected to the internet, you can skip this step by unchecking the option and clicking next.
        5. The next setup screen will deal with installed instances of SQL Servers in the computer you are trying to install.
          • If there is already a software product running, it is a good idea to make another instance. To do this, you should select Add features to an existing instance of SQL Server 2012 and provide a new Server Instance Name (example: MSSQLSERVER2012)
          • If it is a new system, or if the old system does not matter(Consult the IT Department of the institution), then go ahead and Perform a new installation of SQL Server 2012
          • Perform a new installation of SQL Server 2012 is better for our purposes.
        6. The product key, asked on the next screen, should already be available with the installation media.
          • If it is not, please copy paste 748RB-X4T6B-MRM7V-RTVFF-CHC8H
        7. Next,
          • select SQL Server Feature Installation
          • choose all Instance Features along with Management Tools(Basic & Complete)
        8. If you have selected Add features to an existing instance of SQL Server 2012 in the previous steps, you’ll have an option to name a new instance in the following screen. Otherwise, just proceed and complete the setup as usual.
        9. SQL Server 2012 Enterprise should be now properly set up.
    • After opening Management studio, connect to a database engine (SQL Server instance).

      In this step, Please be careful in choosing the correct instance, if there are multiple servers or, instances. Note down the instance name and host name if there are more than one.

    • Now restore the required database with the .bak file. It should be available, for download, in pre-requisites
    • Finally, go to connected database engine > Logins and right-click to create a new login.
      • On the login name, paste IIS APPPOOL\.NET 4.5
      • Give appropriate Server Roles and User Mapping. Generally Read/Write access(ie. db datareader / db datawriter) is enough.

        When selecting User Mapping, it is important to first select the target database

    • The device is now properly prepared for IIS

  2. Open the IIS Manager

    • Press Win+R and type inetmgr then, hit ENTER
    • If this does not open IIS Manager, press Win+R, type optionalfeatures, hit ENTER.
      • Enable IIS along with associated .NET Services.

        • It is very important to check if all the features in the following section are enabled.

          Internet Information Services > World Wide Web Services > Application Development Features > *

          These are expected to be enabled. CGI and FTP are optional for this particular setup

          Click here to view screenshot of relevant optional features

  3. On the IIS Manager, create a frontend and a backend, in each case.

    • Ensure that the binding is taking place properly.
      • Bind with all IP Addresses i.e.* and, specific ports (say) :88 and :99.

    • Add .NET 4.5 to the application pool *The host files for both are associated with .NET 4.5 Framework.

      We have installed .NET 4.5.2 because, at the moment, it is our fronend dependency. .NET 4.5.2 is an in-place replacement for .NET 4.5 and .NET 4.0 . This means, no other frameworks are needed to be installed.

      • The default .NET framework for Windows 7 is .NET 3.5 and it will NOT show any newer .NET framework in the optionalfeatures list even if you install it in your device. However, .NET 4.5 is vital to your Application Pool.
      • To enable .NET 4.5 manually, run this command in cmd.exe with Administrative priviledge.
      %windir%\Microsoft.NET\Framework\v4.0.30319\aspnet_regiis.exe -ir

      The above command runs successfully only when you install .NET 4.5.2 Framework first

      • The .NET 4.5 will still not show up in your Application pool. But you can now manually make an entry like follows:
      Name: .NET 4.5
.NET Framework Version: .NET Framework v4.0.30319
Managed pipeline mode: Integrated
[✓] : Start application pool immediately
      • Then on the host file Basic setting(i.e.IIS Manager > IMS-UI > Basic Setting) you will be finally able to select .NET 4.5

  4. Finally, double check your preparation by hitting the server.

    • If you browse the server, it should display a similar kind of footer information to this:

      Version Information: Microsoft .NET Framework Version:4.0.30319; ASP.NET Version:4.0.30319.34209

      along with a standard 404 Error.

    • If you’re getting Error 500 or 503, something has gone terribly wrong in the previous steps.
    • Also, browse your fronend. It should be able to open but, should not be able to access the server.
