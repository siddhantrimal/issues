Troubleshoot

Here are some common issues that you may face while performing this setup

  1. SQL backup file refuses to restore into the database
    • Ensure that you are using an instance of SQL Server 2012 while connecting to the database-engine from SQL Server Management Studio.
    • If there are multiple named instances, choosing the incorrect one will create this problem.
    • If all else fails, this can be generally resolved by installing SQL Server 2012.
      (Please see pre-requisites).
  2. Can’t see .NET 4.5 in the App Pool
    • This is a problem with older OS (Windows 7)
    • Windows-7 comes preloaded with .NET 3.5 and any newer versions will not show up in optionalfeatures
    • However, it is possible to add a .NET 4.5 entry manually in the App Pool
      (Please see preparing the devive)
  3. IIS Frontend produces errors when I run it
    • Frontend is generally fault tolerant but if there is an error, it is due to the unavailability of .NET 4.5.2, which it requires.
    • Ensure that .NET 4.5.2 is installed
    • Ensure that .NET 4.5 is setup as the Application Pool in Frontend Basic Settings.
  4. ISS Frontend seems to load but the page is stuck at Login screen
    • Frontend is pretty fault tolerant
    • This means there is a problem in the backend or, there might be a problem connecting to SQL database
    • Ensure that IIS Backend Connection Strings are properly configured
    • Ensure that, if there are multiple SQL named instances, your database-engine is connecing to the right one.
    • Ensure that your connection string has the proper source Localhost or, Siddhant-PC\MSSQLSERVER2012 or, .\MSSQLSERVER2012 according to the number of installed instances. You can manually edit this in Web.Config.
    • Ensure that common.services is pointing to the right IP Address. Using Localhost:88 format is often more effective in reducing errors.
    • Ensure that hostfile is getting proper SQL Server Access. Make a manual login entry to fix this. See preparing-the-device
  5. I’m trying to fix by following the steps, but it is not working!
    • One of two things is happening,
      • (Incase of backend problem) Either your SQL Server instance hasn’t restarted since the last change
      • (Incase of frontend problem) Or, your browser is caching up the previous data, making changes in static content (like js and config) not apparent.
    • Fix by opening services(Win+R and type services.msc) and restarting your SQL Server Instance.
    • Fix by clearing cache or, restarting Incognito mode.
