Setting Up

In this segment, we will set up the system to work on a local intranet

Before we begin, please ensure that the MIS is properly functioning in the host computer itself. If it is unable, first configure the binding setup in the IIS itself. Or, check if you have failed to follow the previous steps properly.

Configuring IIS to our needs

  1. Press Win+R and type inetmgr
  2. Go to Basic Settings and ensure Application pool is .NET 4.5 in both frontend and backend
  3. Next, click on Binding Settings and choose the port and IP, with which, you want the local network to interact with the IIS.

    Consult the Network administrator about the type of network they are using. If they are unsure, do not use a static IP Address.

    If the Local Network Router is configured to provide Dynamic IP Address (i.e. DHCP without Address Reservation), you have to choose IP Address as All Unassigned, i.e. *

    If you don’t do this on a dynamic IP network, your computer could take another IP address in the future, making this setup useless.

Modify the Host configuration files

  1. common.services

    • This file is present in the front-end and it tells the host to connect to a particular backend. You can use Localhost followed by a port number to specify connection if the database server exists in the same computer e.g.Localhost:88

  2. Web.config

    • This setting can also be configured firectly from IIS Manager > Connection Strings
    • This file is present in the back-end and it tells the host to connect to a particular database engine or instance. It can also be configured to connect to a database engine running on a connected local computer on the network.

      For our purposes, we use the following setting

      Data Source=Localhost;Initial Catalog=SMS2;Integrated Security=True;

      if there is a seperate SQL Server Instance, during setup, the host-name followed by / and the instance name should be used.

      Data Source=Siddhant-PC\MSSQLSERVER2012;Initial Catalog=SMS2;Integrated Security=True;

      using . instead of the host-name points to the same computer.

      Data Source=.\MSSQLSERVER2012;Initial Catalog=SMS2;Integrated Security=True;

  3. Host configuration is now complete

Setting up the firewall

  1. Press Win+R and type firewall.cpl, then hit ENTER
  2. Click Advanced Settings on the left pane
  3. Create an outbound rule for the two ports that you’ve used for frontend and backend
  4. Similarly, create an inbound rule for the same two ports
  5. Firewall setup is complete
